# Règles design consolidées

- TOKENS COULEUR : utiliser bg-card / bg-surface-muted / bg-surface-elevated / text-foreground / text-muted-foreground / border-border / border-border-strong. NE PLUS écrire bg-white dark:bg-[hsl(var(--card))] → écrire bg-card. NE PLUS étendre la couche de compat (globals.css 405-796).
- FOCUS : un seul token --ring par thème (clair=gold ou foreground/0.55, dark=zinc clair, navy=bleu clair). Toujours focus-visible: (jamais focus: seul). Remplacer les 23 ring-zinc-400 par ring-ring. Outline contraste ≥3:1 (l'actuel hsl(240 5% 65%) échoue).
- MOTION : durées et easings via tokens (--dur-fast 120 / --dur-base 160 / --dur-slow 240 ; --ease-out / --ease-standard / --ease-bounce). Aucune durée/cubic-bezier en dur dans keyframes ou JSX. transition-colors (pas transition-all).
- REDUCED-MOTION : bloc @media prefers-reduced-motion obligatoire + garde-fous JS (confettis, compteur rAF, scrollIntoView). Hover translate/scale sous motion-safe:.
- Z-INDEX : échelle nommée uniquement (dropdown 1000 < sticky 1100 < overlay 1200 < modal 1300 < popover 1400 < toast 1500 < command 1600 < skiplink 2000). Interdit : z-[900/950/1000/1500/2000] arbitraires.
- TYPO : plancher text-2xs (10px) pour tout texte porteur d'info (proscrire 8/9px). Titres leading-tight jamais leading-none. Sous-titres KPI line-clamp-2 (pas truncate qui masque l'ARR). Retirer l'import Fraunces (0 usage, display réelle = Funnel_Display).
- DORÉ : focus, état actif significatif (liseré chip/onglet), valeur à fort enjeu. Variant Button brand doré pour CTA principal. JAMAIS texte doré sur fond doré. Liens en hover:text-gold (pas sky).
- STATUT : sémantique centralisée dans lib/utils.ts avec variantes dark: inline (modèle BADGE_TONE). EN_COURS = sky PARTOUT (corriger blue d'IR/CAA). Fix indigo (Perdu dans l'espace) absent de la compat.
- PASTILLE D'URGENCE : convention unique amber=à traiter, rose=en retard, rien=terminé/inactif. Réutiliser getUrgencyStatus. Tout signal couleur doublé d'un title/aria-label.
- HOVER : hover:bg-muted/60 (dark:hover:bg-white/[0.06]). Interdit hover:bg-zinc-50/50 nu (opacités non remappées en dark). Le hover de ligne doit éclairer la cellule sticky (group/row).
- CARD : une seule (API title/subtitle/action, rounded-xl shadow-card). Supprimer la Card concurrente de ui.tsx. BADGE : une seule API tone sémantique.
- PICKER : la primitive _components/picker.tsx partout (supprimer FactPicker maison + popovers statut dupliqués). ARIA listbox complète (role=option, aria-selected, aria-activedescendant) + React.memo.
- MODALE : ModalShell partagé (portal + backdrop bg-foreground/40 + focus-trap + Esc). aria-labelledby via useId(). Focus initial useRef+useEffect (jamais ref callback inline).
- ICÔNES : lucide-react uniquement. Proscrire glyphes Unicode (flèches de tri, check, croix, retour, sablier) et emojis comme icônes UI.
- TABLE DENSE : thead sticky top-0 z-sticky fond opaque ; tri via ChevronUp/Down lucide ; resize via PointerEvent + poignée focusable clavier ; largeurs lues en lazy useState (pas useEffect → pas de FOUC).
- ÉTAT VIDE / SKELETON : EmptyState partagé (icône cerclée + CTA + reset filtres, distinguer 0-filtré vs 0-base). Skeleton = silhouette exacte du contenu réel (tokens bg-muted), un loading.tsx par sous-route lourde.
- CONFIRMATION : useConfirm variant danger toujours (jamais window.confirm natif). Bulk destructif par colonne aussi confirmé.
- TOASTS : sonner unique, branché sur thème résolu (navy→dark). Supprimer le stack maison Jarvis.
- PERF DATA : Promise.all pour requêtes indépendantes (facturation = 6 séries à paralléliser). Pas de select(*). router.refresh hors chemin optimiste. revalidate ciblé par tag. Badges sidebar event-driven, pas par pathname.
- PERF RENDU : React.memo + clés stables sur lignes/cellules de listes longues ; constante EMPTY pour options vides ; Map pour lookups (pas find() en boucle) ; Recharts lazy (finance) ; ChatBubble lazy.
- A11Y CLAVIER : roving tabindex + role=grid sur la grille obligations ; KeyboardSensor sur le kanban ; focus-trap sur drawers/popovers riches ; cibles tactiles ≥44px (croix p-2, mini-pills ≥10px) ; aria-pressed sur chips/toggles ; aria-current sur onglet actif.

## Notes thèmes

Le système 3 thèmes est cohérent en luminosité mais repose sur une stratégie fragile et contient des trous spécifiques. (1) CLAIR : le focus :focus-visible outline hsl(240 5% 65%) tombe à ~1.8:1 sur blanc (échec WCAG 1.4.11) ; le gradient ambient body est une tache navy hardcodée hors token. (2) DARK : la spec impose 'saturation 0 partout, interdit le bleu nuit' (globals.css:101-103) MAIS le gradient ambient dark introduit hsl(220 50% 30%) = un voile BLEU non voulu — le passer en neutre gris. Le hover natif des <option> n'est pas fiable en Chromium (faux sentiment de contrôle). (3) NAVY : c'est le parent pauvre — navy applique .dark + .navy et n'override QUE les fonds/bordures zinc en bleu, donc TOUT le reste hérite du dark : (a) pas d'override navy pour :focus-visible (focus gris froid sur bleu) ni pour le ring gold/30 du ClientSwitcher (halo invisible) ; (b) les surfaces ambre (matrice obligations, headers de section échéances bg-amber-500/[0.06], badges admin 'En attente') restent brun-chaud greyscale au milieu d'une UI bleu marine — ajouter des overrides navy teintés bleu-doré ou basculer ces accents sur le gold du thème ; (c) les <option> de select n'ont aucun override navy ; (d) le toast sonner ne connaît pas navy (au mieux 'dark'). RÈGLE TRANSVERSE : tout composant central (ClientSwitcher, CommandPalette, pills pipeline, cards) doit porter ses variantes dark: explicites au lieu de dépendre de la compat — la CommandPalette le fait déjà bien (dark:bg-[hsl(var(--surface-elevated))]), le ClientSwitcher pas du tout (bg-white nu) → les deux surfaces de recherche jumelles ont des élévations différentes en navy. Vérifier chaque livrable dans les 3 thèmes côte à côte : un popover navy doit avoir une teinte bleutée cohérente (pas un gris zinc emprunté au dark), et les badges sémantiques (amber/sky/emerald/rose) doivent garder la même densité 'verre dépoli' (bg-*-500/15 dark) plutôt que mélanger aplats opaques (bg-*-100 remappés) et translucides selon les surfaces.
